Project Overview

What is UniHub?

UniHub is a full-stack web application designed to improve how university students collaborate, manage projects, and share knowledge. It simulates a real-world software development environment while solving genuine collaboration challenges faced by student teams.

The platform brings together task management, real-time communication, file sharing, and team coordination into a single, cohesive experience — giving students hands-on exposure to modern development tools and workflows.

Built with a client-server architecture, UniHub uses React on the frontend, Node.js and Express on the backend, and MongoDB for data persistence — with Socket.io powering real-time features.

Core Features

Everything a Student Team Needs

Eight carefully designed features that cover the full lifecycle of a university project — from team formation to final delivery.

🔐
User Management
Secure registration and login with JWT authentication and bcrypt password hashing. Full profile management with bio, skills, and avatars.

Team Collaboration
Create teams, invite members via links, and assign roles (Leader / Member). Organize work across multiple project teams simultaneously.

Mini Jira-style Kanban board. Create, assign, and set deadlines for tasks. Track status from To Do → In Progress → Done.

A central hub showing task summaries, team activity feeds, upcoming deadlines, and progress overviews at a glance.

Advanced feature for managing users, monitoring platform activity, and handling reports. Full oversight for administrators.

Development Process

UniHub Git Workflow Guideline

Team rules and best practices for smooth collaboration using Git and GitHub.

🚫 Important Rules (MUST FOLLOW)
❌ Do NOT push to main branch
❌ Do NOT work directly on main
❌ Do NOT create branches from main
✅ Always work on a separate feature branch
✅ All work must be submitted via Pull Request to dev
✅ Only the team lead reviews and merges
🔄 Standard Workflow (Step-by-Step)
1. Clone the repository
git clone https://github.com/Naol724/UniHub.git
cd UniHub
2. Switch to dev branch
git checkout dev
git pull origin dev
3. Check current branch (VERY IMPORTANT)
Before doing anything, verify your branch:
git branch
You should see:
* dev
main
✅ If not on dev, switch:
git checkout dev
4. Create your own branch
Use a clear branch name based on your task:
git checkout -b feature/your-task-name
Examples:
feature/login-ui
feature/dashboard
feature/auth-api
feature/navbar
5. Work only on your branch
Edit only your assigned files
Do not modify unrelated modules
Do not switch branches while working
6. Save your changes (commit)
git add .
git commit -m "Clear message describing your work"
Example commit messages:
"Add login UI page"
"Implement authentication API"
"Create user model"
7. Push your branch
git push origin feature/your-task-name
8. Create a Pull Request (PR)
On GitHub:
Base branch: dev
Compare branch: your feature branch
❗ Important:
Always target dev
Never create PR to main
9. Review & Merge Process (Handled by Team Lead)
The team lead will:
Review your code
Check for errors
Suggest improvements if needed
Approve the PR
Merge into dev
🔍 Common Mistakes to Avoid
❌ Working on main
❌ Creating branch from main
❌ Pushing without creating a branch
❌ Submitting PR to main
❌ Editing files outside your task scope
✅ Best Practices
✔ Always run: git branch before starting work
✔ Pull latest updates before creating a branch
✔ Keep your branch focused on ONE task
✔ Use clear commit messages
✔ Communicate if your work affects others
🧠 Simple Workflow Summary
Clone → Switch to dev → Check branch → Create feature branch → Work → Commit → Push → PR to dev → Review → Merge
👨‍💼 Role Reminder (Team Lead Responsibility)
As the reviewer and merger, you should:
• Ensure PR is targeting dev
• Check code quality
• Verify no conflicts
• Approve or request changes
• Merge only after successful review
• Reject any PR targeting main
